# Classificação de Stakeholders

Este documento detalha como classificar stakeholders por influência e interesse.

---

## A Matriz

```
                         INTERESSE NO PROJETO
                      Baixo                Alto
                ┌─────────────────┬─────────────────┐
                │                 │                 │
        Alta    │    MANTER       │    ENGAJAR      │
                │   SATISFEITO    │   ATIVAMENTE    │
  INFLUÊNCIA    │                 │                 │
                ├─────────────────┼─────────────────┤
                │                 │                 │
        Baixa   │   MONITORAR     │    MANTER       │
                │                 │   INFORMADO     │
                │                 │                 │
                └─────────────────┴─────────────────┘
```

## Como Avaliar Influência

### Perguntas para determinar nível de influência

| Pergunta | Se SIM | Se NÃO |
|----------|--------|--------|
| Pode aprovar ou vetar o projeto? | Alta | Depende |
| Controla orçamento ou recursos? | Alta | Depende |
| Tem autoridade formal sobre a equipe? | Alta | Depende |
| É respeitado informalmente (influência moral)? | Média-Alta | Depende |
| Pode escalar para alguém com mais poder? | Média | Baixa |
| Suas opiniões são ouvidas por decisores? | Média | Baixa |
| Seria ignorado se discordasse? | Baixa | Média-Alta |

### Indicadores de Alta Influência

- Cargo de diretoria ou C-level
- Controla orçamento do projeto
- É o patrocinador formal
- Outras pessoas pedem aprovação dele
- Pode demitir ou realocar pessoas da equipe
- Tem acesso direto ao CEO/board

### Indicadores de Baixa Influência

- Cargo operacional sem autoridade formal
- Não controla recursos
- Suas opiniões são "consideradas" mas não determinantes
- Não tem acesso a decisores
- Pode ser ignorado sem consequências imediatas

### Armadilha: Influência Informal

**Cuidado:** Algumas pessoas têm alta influência sem cargo formal:
- Especialistas técnicos respeitados
- Funcionários antigos com credibilidade
- Pessoas bem relacionadas politicamente
- Quem tem acesso privilegiado ao patrocinador

> **Dica:** Pergunte "Se essa pessoa disser que o projeto é ruim, o que acontece?"

## Como Avaliar Interesse

### Perguntas para determinar nível de interesse

| Pergunta | Se SIM | Se NÃO |
|----------|--------|--------|
| Será diretamente afetado pelo resultado? | Alto | Depende |
| O projeto muda o trabalho diário dele? | Alto | Depende |
| Tem metas pessoais ligadas ao projeto? | Alto | Médio |
| Pergunta sobre o projeto proativamente? | Alto | Baixo |
| Participa de reuniões sem ser obrigado? | Alto | Baixo |
| Se o projeto falhar, ele perde algo? | Alto | Baixo |
| Se o projeto der certo, ele ganha algo? | Alto | Baixo |

### Indicadores de Alto Interesse

- Vai usar o sistema diariamente
- Seu bônus/promoção depende do resultado
- Foi ele quem solicitou o projeto
- Sua área é a principal beneficiada
- Participou da concepção da ideia

### Indicadores de Baixo Interesse

- Não será afetado diretamente
- Projeto é tangencial à sua área
- Não comparece em reuniões
- Delega participação para subordinados
- Responde com atraso ou superficialmente

### Armadilha: Interesse Oculto

**Cuidado:** Algumas pessoas têm interesse que não demonstram:
- Medo de perder poder ou relevância
- Preocupação com mudança de processos
- Interesse negativo (quer que falhe)
- Interesse político (quer crédito ou quer evitar culpa)

> **Dica:** Pergunte "O que essa pessoa ganha ou perde com o projeto?"

## Classificação Passo a Passo

### Passo 1: Liste todos os stakeholders

Inclua:
- Quem solicitou
- Quem financia
- Quem aprova
- Quem usa
- Quem é afetado
- Quem implementa
- Quem dá suporte

### Passo 2: Avalie cada um

Para cada stakeholder, preencha:

| Stakeholder | Pode vetar? | Controla recursos? | É afetado? | Tem interesse ativo? |
|-------------|-------------|-------------------|------------|---------------------|
| {nome} | Sim/Não | Sim/Não | Sim/Não | Sim/Não |

### Passo 3: Classifique

| Combinação | Classificação |
|------------|---------------|
| Pode vetar OU controla recursos + Demonstra interesse ativo | **Engajar Ativamente** |
| Pode vetar OU controla recursos + Pouco interesse demonstrado | **Manter Satisfeito** |
| Não pode vetar + É muito afetado OU tem interesse ativo | **Manter Informado** |
| Não pode vetar + Pouco afetado + Pouco interesse | **Monitorar** |

### Passo 4: Valide

- Se todos são "Engajar Ativamente" → reavalie, não é possível
- Se ninguém é "Engajar Ativamente" → falta patrocinador, risco alto
- Se muitos são "Monitorar" → pode estar ignorando stakeholders importantes

## Dinâmica: Stakeholders Mudam

### Quando Reclassificar

| Evento | O que verificar |
|--------|-----------------|
| Mudança organizacional | Quem ganhou/perdeu poder? |
| Novo executivo | Quais são suas prioridades? |
| Crise no projeto | Quem está mais atento agora? |
| Sucesso parcial | Quem quer se associar ao sucesso? |
| Fracasso parcial | Quem quer se distanciar? |
| Mudança de escopo | Quem é mais/menos afetado? |

### Movimentos Comuns

```
MANTER SATISFEITO → ENGAJAR ATIVAMENTE
(quando o projeto vira prioridade do executivo)

MONITORAR → MANTER INFORMADO
(quando descobrem que serão afetados)

ENGAJAR ATIVAMENTE → MANTER SATISFEITO
(quando delegam para subordinado)

MANTER INFORMADO → ENGAJAR ATIVAMENTE
(quando usuário vira champion do projeto)
```

## Erros Comuns na Classificação

### Erro 1: Confundir cargo com influência

**Errado:** "É gerente, então tem alta influência"

**Correto:** Avalie influência REAL sobre ESTE projeto. Um gerente pode ter baixa influência se o projeto não é da área dele.

---

### Erro 2: Assumir que interesse é positivo

**Errado:** "Ele tem alto interesse, vai nos ajudar"

**Correto:** Interesse pode ser negativo. Alguém pode ter alto interesse em BLOQUEAR o projeto.

---

### Erro 3: Ignorar stakeholders "baixos"

**Errado:** "São só usuários, baixa influência"

**Correto:** Usuários podem sabotar a adoção. Influência coletiva é real.

---

### Erro 4: Classificação estática

**Errado:** Mapear uma vez e nunca revisar

**Correto:** Revisar a cada fase ou quando houver mudanças organizacionais.
